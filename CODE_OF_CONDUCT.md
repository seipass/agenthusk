# Code of Conduct

AgentHusk contributors and users should keep project spaces professional, specific, and safe.

## Expected behavior

- Discuss technical tradeoffs directly and respectfully.
- Assume good intent while asking for reproducible evidence.
- Use synthetic values in issues, pull requests, and test fixtures.
- Respect privacy. Do not publish report files, real credentials, or sensitive paths.
- Keep security disclosures private until maintainers coordinate a response.

## Unacceptable behavior

- Harassment, threats, discriminatory language, or personal attacks.
- Publishing another person's private information.
- Posting real credentials, sensitive paths, or unreviewed AgentHusk reports.
- Using project spaces to pressure maintainers into unsafe disclosure.

## Reporting

For a conduct concern, contact a maintainer privately through the repository profile. For a vulnerability, follow [SECURITY.md](SECURITY.md).

Maintainers may remove content or restrict participation when needed to protect contributors and users.
